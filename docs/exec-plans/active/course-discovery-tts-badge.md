# 课程发现页「可听课」徽章 — ExecPlan

> 详细设计见 [docs/superpowers/specs/2026-07-06-course-discovery-tts-badge-design.md](../../superpowers/specs/2026-07-06-course-discovery-tts-badge-design.md)。本计划是其执行蓝图,遵循 `PLANS.md`,承接已完成的 [homepage-course-discovery](./../completed/homepage-course-discovery.md)。

## Purpose / Big Picture

在首页课程发现卡片上,用一个**颜色独特的「可听课」绿色徽章**标识已开启 TTS(`tts_enabled == 1`)的课程,让访客一眼区分「可听课 / 仅读课」。本期只做**展示**,不做筛选。

- 后端:`get_published_course_catalog` 返回项补一个 `tts_enabled` 布尔(零迁移、零新查询,字段已在 `PublishedShifu` 上)。
- 前端:`Badge` 新增 `success` 绿色变体 + `globals.css` 补 `--success` 色变量;`CourseCard` 加一个徽章分支(**与登录态无关,匿名也可见**);三语 i18n。
- 不改:`/c` 学习页、创作者端配置、「我的课程」列表、发现页筛选控件。

## Decision Log

- **落点**:只做发现层「展示徽章」,不做筛选(Tab/Toggle/后端过滤参数)。原因:用户明确选“仅徽章,不做筛选”,YAGNI。
- **徽章颜色**:新增 `Badge` `success`(绿)变体 + `globals.css` `--success`/`--success-foreground`。原因:现有 4 变体已被占用(default=自有、secondary=已购入、outline=学习中/已完成/归档、destructive=危险语义),用户要求“不同颜色”,绿色正向且独特。
- **判定标准**:`tts_enabled == 1` 单字段,不额外校验 provider/voice。原因:课程保存 TTS 时已走 `validate_tts_settings_strict`,开启者配置基本齐全;在发现层重复校验是 N+1 风险且收益小。
- **登录态**:徽章**与登录态无关**(匿名、isGuest、登录用户均显示),区别于 owner/purchased 等“需登录”徽章。原因:`tts_enabled` 是课程公开属性,对访客也有价值。
- **字段命名**:后端直出 `tts_enabled`(snake_case,沿用数据库列名),前端 `CourseItem.tts_enabled` 同名承接,不引入 `is_` 前缀映射。原因:与 `ShifuDetailDto` 及同返回项风格一致。
- **i18n key**:`common.core.courseAudioAvailable`,三语 en-US=`Audio Available`、zh-CN=`可听课`、fr-FR=`Audio disponible`。
